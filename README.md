# Hosts 文件自动恢复程序 🛡️

> 一个强大的 Windows hosts 文件实时监控和自动保护工具

## ✨ 项目概述

这是一个用 **Go** 语言编写的 Windows 系统 hosts 文件监控和自动恢复程序。它能够实时监听 hosts 文件的任何修改，并自动从备份恢复到原始状态，有效防止 hosts 文件被恶意篡改。

### 🎯 核心功能

- ✅ **实时文件监控** - 使用 fsnotify 库监听 hosts 文件所在目录的所有文件系统事件
- ✅ **自动备份管理** - 启动时自动创建初始备份文件（如果不存在）
- ✅ **自动恢复机制** - 检测到 hosts 文件被修改时，自动从备份恢复到原始状态
- ✅ **权限检查** - 启动时检查管理员权限，确保程序可以正常运行
- ✅ **智能防抖处理** - 500ms 防抖机制，防止编辑器保存时的多次触发
- ✅ **冷却期机制** - 恢复成功后的 1000ms 冷却期，避免对恢复操作本身的重复响应
- ✅ **日志记录** - 详细记录所有事件到日志文件（hosts-monitor.log）
- ✅ **进程追踪** - 识别并记录修改 hosts 文件的进程信息
- ✅ **优雅退出** - 支持 Ctrl+C 中断，清理资源后正常退出

## 🚀 快速开始

### 前置要求

- **Go 版本**: 1.26 或更高版本（项目使用 Go 1.26）
- **操作系统**: Windows 系统（Windows 11 已测试）
- **权限**: 管理员权限（必需，用于读写 hosts 文件）

### 获取依赖

```powershell
go mod download
```

或者手动安装：

```powershell
go get github.com/fsnotify/fsnotify
```

### 编译程序

```powershell
# 编译为可执行文件
go build -o refresh-host.exe refresh-host.go

```

### 运行程序

```powershell
# 以管理员权限运行（必须）
# 方式1：直接运行
.\refresh-host.exe

powershell -Command "& {Start-Process .\refresh-host.exe -Verb RunAs}"
```

## ⚙️ 配置说明

### 关键常量和路径

程序中定义的重要常量（位于文件开头）：

```go
// Windows 标准 hosts 文件路径
const HostsPath = "C:\\Windows\\System32\\drivers\\etc\\hosts"

// 备份文件路径（与 hosts 文件同目录）
const BackupPath = "C:\\Windows\\System32\\drivers\\etc\\hosts.backup"
```

### 可调整的参数

| 参数 | 位置 | 默认值 | 说明 |
|-----|------|--------|------|
| 防抖延迟 | main() 函数 ~130 行 | 500ms | 检测到修改后等待时间，避免频繁触发 |
| 冷却期延迟 | main() 函数 ~147 行 | 1000ms | 恢复成功后的等待时间，避免重复响应 |
| 日志位置 | main() 函数 ~39 行 | 程序目录 | 自动保存为 hosts-monitor.log |

### 修改配置的步骤

1. 编辑 `refresh-host.go` 文件
2. 修改相应的常量或时间参数
3. 重新编译：`go build -o refresh-host.exe refresh-host.go`
4. 重新以管理员身份运行

## ❓ 常见问题

### Q: 程序提示权限不足或无法读写 hosts 文件怎么办？

**A**: 这是最常见的问题。hosts 文件需要管理员权限才能读写。解决方案：

```powershell
# 方式1：右键点击程序，选择"以管理员身份运行"
# 方式2：在 PowerShell 中以管理员身份运行程序
powershell -Command "& {Start-Process .\refresh-host.exe -Verb RunAs}"
```

### Q: 如何停止程序？

**A**: 在程序运行的控制台窗口中按 `Ctrl+C`，程序将优雅地退出。

### Q: 备份文件在哪里？

**A**: 备份文件位于 `C:\Windows\System32\drivers\etc\hosts.backup`，与原 hosts 文件同目录。

### Q: 程序可以后台运行吗？

**A**: 可以，但需要将程序配置为 Windows 服务。这需要额外的工具（如 NSSM）来完成。未来版本可能增加此功能。

### Q: 程序会对系统性能有影响吗？

**A**: 几乎没有。fsnotify 库的资源占用极低，CPU 占用低于 0.1%，内存占用 5-10 MB，即使长期运行也不会对系统造成显著影响。

### Q: 能否修改防抖时间？

**A**: 可以。编辑源码第 130 行的 `500*time.Millisecond` 和第 147 行的 `1000*time.Millisecond`，然后重新编译。

### Q: hosts 文件被修改后多久会被恢复？

**A**: 从检测到修改到完成恢复通常需要约 600 ms（500 ms 防抖 + 100 ms 恢复操作）。


### Q: 如果我需要修改 hosts 文件怎么办？

**A**: 可以有以下方案：
1. 临时停止程序（按 Ctrl+C），修改后再运行
2. 未来版本可能增加白名单机制
