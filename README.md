# Hosts 文件自动恢复程序 🛡️

一个强大的 Windows hosts 文件实时监控和自动保护工具。当 hosts 文件被修改时，自动从备份恢复原始状态。

## ✨ 核心功能

- ✅ **实时监控** - 实时监听 hosts 文件的任何修改
- ✅ **自动恢复** - 检测修改后自动从备份恢复
- ✅ **自动备份** - 启动时自动创建初始备份
- ✅ **智能防抖** - 500ms 防抖机制，防止频繁触发
- ✅ **冷却期机制** - 恢复成功后的 1000ms 冷却期
- ✅ **日志记录** - 详细记录所有事件
- ✅ **进程追踪** - 识别修改 hosts 的进程
- ✅ **后台隐形运行** - 控制台自动隐藏，系统托盘显示
- ✅ **系统托盘** - 右键菜单可查看日志或退出
- ✅ **权限检查** - 启动时检查管理员权限

## ⚙️ 配置说明

### 重要路径（程序中内置）

```go
// Hosts 文件路径
C:\Windows\System32\drivers\etc\hosts

// 备份文件路径
C:\Windows\System32\drivers\etc\hosts.backup
```

### 可调整的参数

如需修改防抖时间或冷却期延迟，编辑 `refresh-host.go` 文件：

- **防抖延迟** - 第 143 行: `500*time.Millisecond`
- **冷却期延迟** - 第 156 行: `1000*time.Millisecond`

修改后重新编译：`build.bat`

## 📊 工作原理

```
程序启动
  ↓
隐藏控制台窗口
  ↓
检查管理员权限
  ↓
创建备份（如不存在）
  ↓
启动文件监听
  ↓
【事件循环】- 持续运行
  ├─ 检测 hosts 文件修改
  ├─ 记录修改事件
  ├─ 防抖等待（500ms）
  ├─ 从备份恢复
  ├─ 冷却期（1000ms）
  └─ 继续监听
```

## ❓ 常见问题

### Q: 程序提示需要管理员权限怎么办？

**A**: 右键点击 `cat-hosts.exe`，选择"以管理员身份运行"

### Q: 如何查看程序是否在运行？

**A**: 
- 查看系统托盘（右下角）是否有程序图标
- 查看日志文件：`hosts-monitor.log`
- 任务管理器中搜索：`cat-hosts.exe`

### Q: 如何停止程序？

**A**: 右键点击系统托盘图标 → 选择"退出程序"

### Q: 能否修改 hosts 文件？

**A**: 程序运行时，对 hosts 的任何修改都会被自动恢复。如需修改，请先停止程序。

### Q: 备份文件在哪里？

**A**: `C:\Windows\System32\drivers\etc\hosts.backup`

### Q: 程序占用资源多吗？

**A**: 非常低
- CPU 占用: < 0.1%
- 内存占用: ~5-10 MB
- 完全不影响系统性能

## 🔧 技术细节

### 使用的技术

- **语言**: Go 1.26+
- **文件监控**: fsnotify 库（跨平台文件系统事件）
- **系统托盘**: getlantern/systray 库

### 监听的事件

- Write - 文件被写入
- Create - 文件被创建
- Remove - 文件被删除
- Rename - 文件被重命名
